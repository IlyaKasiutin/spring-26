## 1. Описание метода

**Random Forest** — ансамблевый метод машинного обучения, предложенный Л. Брейманом (2001). Основная идея: объединить большое число деревьев решений, каждое из которых обучается на случайной подвыборке данных, а при построении каждого узла рассматривает случайное подмножество признаков. За счёт двух источников случайности деревья получаются различными, и их совокупный голос даёт значительно меньшую дисперсию ошибки, чем любое одиночное дерево.

### Алгоритм построения

Для каждого дерева $t = 1, \ldots, T$:

1. Сформировать **бутстрэп-выборку** $\mathcal{B}_t$ — случайная выборка с возвращением объёма $n$ из исходных $n$ объектов. В среднем ≈ 63.2 % объектов попадают в $\mathcal{B}_t$, остальные ≈ 36.8 % составляют **OOB-подмножество** $\overline{\mathcal{B}}_t$.
2. Обучить дерево на $\mathcal{B}_t$. При каждом сплите узла рассматривать только случайные `max_features` признаков из $p$ (стандартно: $\lfloor\sqrt{p}\rfloor$).

Предсказание — мажоритарное голосование всех деревьев:

$$\hat{y} = \mathrm{mode}\{h_t(\mathbf{x}),\; t=1,\ldots,T\}$$

### Out-of-Bag оценка (OOB)

Каждый объект $i$ не участвует в обучении части деревьев. Предсказание по OOB:

$$\hat{y}_i^{\mathrm{OOB}} = \mathrm{mode}\{h_t(\mathbf{x}_i) : i \notin \mathcal{B}_t\}$$

OOB accuracy — несмещённая оценка обобщающей способности, не требующая отдельной валидационной выборки.

### Важность признаков OOB$^j$ (пермутационная)

Для каждого дерева $t$ и признака $j$:

1. Вычислить базовую OOB-точность: $a_t = \mathrm{acc}(y_{\overline{\mathcal{B}}_t},\; h_t(X_{\overline{\mathcal{B}}_t}))$
2. Случайно перемешать значения признака $j$ в OOB-подмножестве → $a_t^{(j)}$
3. $\Delta_t^{(j)} = a_t - a_t^{(j)}$

$$\mathrm{Imp}^j = \frac{1}{T}\sum_{t=1}^{T}\Delta_t^{(j)}$$

Чем больше падает точность при перемешивании, тем важнее признак для модели.

## 3. Реализация

Класс `RandomForest` реализован на Python/NumPy и унаследован от `BaseEstimator, ClassifierMixin` для совместимости со sklearn API.

| Метод | Описание |
|---|---|
| `fit(X, y)` | Обучение ансамбля, накопление OOB-голосов, вычисление `oob_score_` |
| `predict(X)` | Мажоритарное голосование деревьев |
| `oob_feature_importance(X, y)` | Пермутационная важность OOB$^j$ |

В качестве базового алгоритма использован `sklearn.tree.DecisionTreeClassifier`.

### Подбор гиперпараметров

Использован `GridSearchCV` из sklearn с кастомным scorer'ом `oob_scorer`, который возвращает `estimator.oob_score_` — OOB accuracy, накопленную во время `fit()`. Поскольку OOB является внутренней оценкой модели, кросс-валидация не нужна: передаётся один фиктивный фолд.

Сетка поиска:

| Параметр | Значения |
|---|---|
| `n_estimators` | 50, 100, 200 |
| `max_depth` | None, 5, 10 |
| `max_features` | `'sqrt'`, `'log2'` |
| `min_samples_split` | 2, 5 |

Всего 36 комбинаций.

## 4. Результаты экспериментов

### 4.1 Подбор гиперпараметров (Grid Search по OOB)

Лучшая комбинация параметров:

| Параметр | Значение |
|---|---|
| `n_estimators` | 100 |
| `max_depth` | None |
| `max_features` | `'sqrt'` |
| `min_samples_split` | 2 |
| **OOB accuracy** | **0.9692** |

`max_depth=None` означает, что деревья растут без ограничения глубины. Для Random Forest это стандартная и теоретически обоснованная практика: каждое дерево намеренно переобучается под свою бутстрэп-выборку, а ансамблирование компенсирует высокую дисперсию. Grid Search честно сравнил все варианты (None, 5, 10) по OOB и выбрал None как наилучший.

### 4.2 Важность признаков OOB$^j$

| Ранг | Признак | Важность |
|:---:|---|:---:|
| 1 | worst area | 0.1501 |
| 2 | worst radius | 0.1366 |
| 3 | worst concave points | 0.1241 |
| 4 | worst perimeter | 0.1174 |
| 5 | mean concave points | 0.0555 |
| 6 | worst concavity | 0.0522 |

Доминируют признаки группы `worst` — наихудшие (максимальные) значения геометрических характеристик опухоли. Это соответствует медицинской интерпретации: злокачественные опухоли характеризуются выраженными экстремальными отклонениями формы и размера клеточных ядер.

## 5. Сравнение с эталонной реализацией (scikit-learn)

Обе модели обучены с одинаковыми гиперпараметрами, найденными Grid Search.

| Метрика | Наш Random Forest | sklearn RandomForestClassifier |
|---|:---:|:---:|
| **Accuracy (test)** | **0.9561** | **0.9561** |
| OOB score | 0.9692 | 0.9538 |
| Время обучения, с | 0.151 | 0.152 |

**Точность на тесте идентична** — 95.61 %, что подтверждает корректность реализации.

**OOB score** незначительно расходится (0.9692 vs 0.9538) из-за разных внутренних случайных состояний при генерации бутстрэп-выборок — оба значения находятся в пределах нормального разброса метрики.

**Время обучения** практически одинаковое, поскольку узкое место в обоих случаях — `DecisionTreeClassifier` из sklearn.

## 6. Выводы

1. **Реализация корректна.** Точность на тестовой выборке совпадает с эталонной реализацией sklearn (0.9561).
2. **OOB — полноценная замена кросс-валидации** при подборе гиперпараметров: без выделения валидационного набора Grid Search нашёл параметры, дающие высокое качество на тесте.
3. **`max_depth=None` — ожидаемый результат** для Random Forest: глубокие деревья с высокой дисперсией хорошо усредняются ансамблем.
4. **Пермутационная важность OOB$^j$** содержательна: топ признаков (`worst area`, `worst radius`, `worst concave points`) согласуется с клинической интерпретацией задачи диагностики рака.
5. **Производительность** нашей реализации сопоставима со sklearn, поскольку вычислительное ядро (дерево решений) заимствовано из sklearn. Для полностью «сырой» реализации дерева на NumPy потребовался бы Cython/Numba для достижения сопоставимой скорости.
